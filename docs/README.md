Цели и задачи тестирования

Основной целью тестирования является проверка корректности работы функций расчета площади и периметра геометрических фигур (круг, квадрат, прямоугольник, треугольник). Задачи тестирования включают:

Проверку правильности математических расчетов для валидных входных данных
Валидацию обработки некорректных входных данных (отрицательные значения)
Проверку соблюдения математических ограничений (неравенство треугольника)
Обеспечение надежности и стабильности работы функций
Описание тестируемого продукта

Продукт представляет собой библиотеку геометрических функций на языке Python, содержащую модули для расчета площади и периметра основных геометрических фигур:

circle.py: функции для работы с кругом (площадь и длина окружности)
square.py: функции для работы с квадратом (площадь и периметр)
rectangle.py: функции для работы с прямоугольником (площадь и периметр)
triangle.py: функции для работы с треугольником (площадь и периметр)
Все функции должны корректно обрабатывать входные параметры и выбрасывать исключения при некорректных данных.

## CI/CD Pipeline

Проект использует GitHub Actions для автоматического тестирования. При каждом push в ветку main и при создании pull request'ов автоматически запускается CI/CD pipeline, который:

- Запускает unit-тесты на двух операционных системах: Ubuntu и Windows
- Тестирует код на различных версиях Python (3.8, 3.9, 3.10, 3.11)
- Проверяет покрытие кода тестами
- Гарантирует качество кода перед слиянием изменений

### Структура CI/CD

Workflow файл находится в `.github/workflows/main.yml` и включает:
- Автоматический запуск при push/pull request
- Матричное тестирование на нескольких ОС и версиях Python
- Установку зависимостей
- Запуск тестов с подробным выводом
- Опциональную проверку покрытия кода

Область тестирования

Тестированию подлежат следующие компоненты:

Функции расчета площади для всех геометрических фигур
Функции расчета периметра для всех геометрических фигур
Валидация входных параметров (проверка на отрицательные значения)
Специфические ограничения (неравенство треугольника для triangle.py)
Стратегия тестирования

Используется unit-тестирование с применением фреймворка unittest. Стратегия включает:

Функциональное тестирование: проверка корректности расчетов для допустимых значений
Тестирование граничных условий: проверка поведения при нулевых значениях
Негативное тестирование: проверка обработки исключительных ситуаций
Тестирование математических ограничений: проверка соблюдения геометрических законов
Тесты организованы в классы TestCircle, TestRectangle, TestSquare, TestTriangle, наследующие от unittest.TestCase.

Критерии приемки

Тестирование считается успешным, если:

Все unit-тесты проходят без ошибок
Функции корректно рассчитывают площадь и периметр для положительных значений
При передаче отрицательных значений выбрасывается ValueError
Для треугольника проверяется неравенство треугольника
Код соответствует стандартам PEP 8
Все функции имеют документацию
Ожидаемые результаты

По итогам тестирования ожидаются:

Отчет о прохождении всех тестов (100% success rate)
Подтверждение корректности математических расчетов
Фиксация всех найденных багов в системе отслеживания
Метрики покрытия кода тестами (стремление к 100%)
Документация по найденным дефектам и их исправлению

Unit-тестирование

Запуск тестов

Для запуска unit-тестов выполните команду:

python -m unittest tests.py

test_area_zero_radius: проверка площади с нулевым радиусом
test_area_negative_radius: проверка исключения при отрицательном радиусе
test_perimeter_positive_radius: проверка периметра с положительным радиусом
test_perimeter_zero_radius: проверка периметра с нулевым радиусом
